# ARB-022.2.2-FIX-001 — Kafka Publisher Review Fixes

## Source Findings

| ID | Finding | Disposition |
|----|---------|-------------|
| A2 | Publisher bean not guarded by `@ConditionalOnBean(OutboundRoutingStrategy.class)` — could fail at startup if no strategy is registered | Fixed |
| H2 | No ArchUnit rule preventing Kafka dependencies from leaking into `platform.messaging.outbox` / `inbox` contracts | Fixed |
| R2 | `OutboundRoutingStrategy` Javadoc described a "logical destination" abstraction while the adapter uses the returned value directly as the Kafka topic | Fixed |
| T2 | No test verifying that a routing-strategy exception propagates and `KafkaTemplate.send()` is never called | Fixed |

---

## Changes Applied

### A2 — `@ConditionalOnBean(OutboundRoutingStrategy.class)`

**File:** `platform/.../spring/KafkaPublisherAutoConfiguration.java`

Added `@ConditionalOnBean(OutboundRoutingStrategy.class)` to the `kafkaOutboundMessagePublisher` bean method. The publisher is now only registered when both a `KafkaTemplate` and a routing strategy bean are present. Updated class-level Javadoc to document this condition.

### H2 — ArchUnit rule

**File:** `platform/src/test/java/.../PlatformArchitectureTest.java`

Added `messaging_contracts_must_not_depend_on_kafka()` — asserts that classes in `platform.messaging.outbox` and `platform.messaging.inbox` do not depend on `org.springframework.kafka..` or `org.apache.kafka..`. Mirrors the existing `messaging_domain_must_not_depend_on_jpa_or_spring` rule in pattern and scope.

### R2 — `OutboundRoutingStrategy` Javadoc

**File:** `platform/.../messaging/outbox/OutboundRoutingStrategy.java`

Replaced the "logical destination / domain-level routing identifier" framing with accurate description: the returned string is used directly as the Kafka topic name by `KafkaOutboundMessagePublisher`. Updated both the interface and the method Javadoc. No logical-to-physical translation layer introduced.

### T2 — Routing strategy failure propagation test

**File:** `platform/.../messaging/kafka/outbound/KafkaOutboundMessagePublisherTest.java`

Added `routing_strategy_failure_propagates_and_send_is_never_called()` — stubs `routingStrategy.resolveDestination()` to throw `IllegalStateException`, asserts `publish()` propagates the exception, and verifies `kafkaTemplate.send()` is never invoked. Added `assertThatThrownBy` and `never()` imports.
