# ARB-022.3.2 — Sequential Pending Message Dispatch

## Service

**Class:** `SequentialPendingDispatchService`  
**Package:** `com.arbitrier.platform.messaging.outbox.application`

Dependencies (ports and application services only, no Kafka):
- `OutboxRepository` — `findPending(int limit)`
- `DispatchOutboxMessageService` — `dispatch(OutboxEvent)`

## Public API

```java
CompletionStage<Void> dispatchPending(int limit)
```

| Input | Behaviour |
|-------|-----------|
| `limit < 0` | Throws `IllegalArgumentException` immediately |
| `limit == 0` | Returns already-completed stage; repository not accessed |
| No pending messages | Returns already-completed stage; dispatcher not called |
| Messages present | Dispatches each sequentially; returns stage representing full batch |

## Sequential Algorithm

A left-fold builds the stage chain synchronously at call time:

```java
CompletionStage<Void> chain = CompletableFuture.completedFuture(null);
for (OutboxEvent event : pending) {
    chain = chain.thenCompose(ignored -> dispatcher.dispatch(event));
}
return chain;
```

Each `thenCompose` fires only after the previous stage completes normally. No executor, no parallel stream, no virtual thread pool. Messages are dispatched strictly one after another.

## Failure Semantics

If any `dispatcher.dispatch(event)` returns a failed stage (or throws synchronously), the `thenCompose` chain for all subsequent messages is never invoked — `CompletionStage` composition guarantees that a failing stage propagates the exception and skips downstream handlers. The returned stage completes exceptionally with the first failure's exception. Remaining messages in the batch are not dispatched.

This is a deliberate stop-on-first-failure policy. Future retry and DLQ handling belong to later slices.

## Ordering Guarantees

Messages are dispatched in the order returned by `OutboxRepository.findPending(limit)`, which is `occurredAt` ascending (oldest first, per ARB-022.3.1). The sequential fold preserves this order: `pending.get(0)` is dispatched first, `pending.get(n-1)` last.
