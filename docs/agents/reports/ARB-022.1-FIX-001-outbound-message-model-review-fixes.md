## Source Findings

| ID | Severity | Finding | Disposition |
|----|----------|---------|-------------|
| G1 | Warning | Documentation stated "exactly once" delivery for commands ‚Äî incorrect; outbox guarantees at-least-once transport, not exactly-once | Fixed |

## Changes Applied

### Documentation ‚Äî G1

**File:** `docs/agents/reports/ARB-022-1-outbound-message-model.md`

Replaced:
> "Commands must be delivered reliably, exactly once, in the same transactional scope as the saga state change."

With:
> "Commands must be written to the outbox atomically with the saga state change, ensuring eventual at-least-once delivery; idempotent processing at the recipient guards against duplicates."

Also updated `CLAUDE.md` `@EntityScan` example to show an entity class (`ServiceEntity.class`) rather than a repository interface.

### Configuration ‚Äî S1 (spring.jpa.packages removed)

Removed the `spring.jpa.packages` block from `application.yml` in all four services. `@EntityScan` on the `*PersistenceConfiguration` class is now the sole entity scanning mechanism.

| File |
|------|
| `server/order-service/src/main/resources/application.yml` |
| `server/inventory-service/src/main/resources/application.yml` |
| `server/credit-service/src/main/resources/application.yml` |
| `server/orchestrator-service/src/main/resources/application.yml` |

### Configuration ‚Äî S1b (EntityScan marker classes)

Replaced repository interface markers with entity class markers in `@EntityScan`. The scanned package is unchanged ‚Äî both the repository interface and the entity class live in `adapter.outbound.persistence`.

| File | Old marker | New marker |
|------|-----------|-----------|
| `order-service/‚Ä¶/OrderPersistenceConfiguration.java` | `SpringDataOrderRepository.class` | `OrderEntity.class` |
| `inventory-service/‚Ä¶/InventoryPersistenceConfiguration.java` | `SpringDataStockReservationRepository.class` | `StockReservationEntity.class` |
| `credit-service/‚Ä¶/CreditPersistenceConfiguration.java` | `SpringDataCreditReservationRepository.class` | `CreditReservationEntity.class` |
| `orchestrator-service/‚Ä¶/OrchestratorPersistenceConfiguration.java` | `SpringDataSagaRepository.class` | `SagaEntity.class` |

### Test ‚Äî T2 (COMMAND round-trip)

**File:** `server/platform/src/test/java/‚Ä¶/outbox/adapter/JpaOutboxRepositoryAdapterTest.java`

Added `command_nature_survives_round_trip()` ‚Äî saves an `OutboxEvent` with `MessageNature.COMMAND`, retrieves it via `findPending()`, and asserts the recovered `messageNature` is `COMMAND`. Added supporting `createPendingCommand()` helper.

## Remaining Risk

None introduced by this fix. The entity scanning change is semantically equivalent ‚Äî `@EntityScan(basePackageClasses = X)` scans the package containing `X`, which is `adapter.outbound.persistence` whether `X` is the repository interface or the entity class.
