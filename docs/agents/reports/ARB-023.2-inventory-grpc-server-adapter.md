# ARB-023.2 — Inventory gRPC Server Adapter

## Context

ARB-023 introduces the first executable business entry flow in Arbitrier. ARB-023.2 implements
the Inventory Service gRPC server adapter for `InventoryAvailabilityService.CheckAvailability`,
turning the ARB-023.1 wire contract into an executable capability.

The operation is read-only. No stock is reserved, decremented, locked, or mutated.

---

## Summary

A complete hexagonal gRPC server slice was implemented in the `inventory-service` module:
application query model, use case interface and service, outbound query port, gRPC inbound adapter
with three supporting mappers, and an in-memory test adapter. 29 new tests were added:
14 use-case unit tests, 12 gRPC adapter unit tests, and 3 in-process gRPC integration tests.

The Spring context loads with the gRPC server registering
`arbitrier.inventory.v1.InventoryAvailabilityService` on port 9091.

## Application Boundary

The gRPC adapter depends on the application input port:

```text
InventoryAvailabilityGrpcService
    ↓ CheckInventoryAvailabilityUseCase (interface)
    ↓ CheckInventoryAvailabilityService (impl)
    ↓ InventoryAvailabilityQueryPort (interface)
    ↓ ConfigurableInventoryAvailabilityQueryAdapter (test) / [production impl TBD]
```

No protobuf type crosses into the application or domain layer.

---

## Query-Port Decision

The existing domain uses String SKU (no `ProductId` type exists). The new
`InventoryAvailabilityQueryPort` uses `String productId` throughout for consistency with the
existing domain vocabulary. No new domain value object was introduced.

`InventoryAvailabilityQueryPort.findAvailability(Collection<String>)` returns
`Map<String, InventoryAvailabilitySnapshot>`. Absence of a key means `PRODUCT_NOT_FOUND`.
This avoids N+1 queries when the implementation fetches all products in one operation.

---

## Inventory Quantity Semantics

The existing Inventory Service database schema (V1 migration) contains only reservation tables
(`stock_reservations`, `stock_reservation_lines`, `stock_allocations`). There is no stock-levels
table. No production JPA adapter for stock availability exists; the existing `WarehouseAllocationPort`
and `StockAvailabilityPort` also lack production implementations.

Following the established project pattern, `InventoryAvailabilityQueryPort` is defined as an
outbound port without a production JPA adapter. The integration proof uses
`ConfigurableInventoryAvailabilityQueryAdapter` (in-memory). A production adapter requires
a database migration adding a stock-levels table, deferred to a future task.

## Protobuf Mappings

### Request mapping

| Protobuf field | Application field |
|---|---|
| `request_id` | `InventoryAvailabilityQuery.requestId` |
| `items[].product_id` | `RequestedInventoryItem.productId` (String) |
| `items[].quantity` | `RequestedInventoryItem.quantity` (int) |

### Response mapping

| Application result | Protobuf response |
|---|---|
| `Available` | `status = AVAILABILITY_STATUS_AVAILABLE`, `unavailable_items = []` |
| `Unavailable` | `status = AVAILABILITY_STATUS_UNAVAILABLE`, `unavailable_items = [...]` |

### Unavailable reason mapping

| Application enum | Protobuf enum |
|---|---|
| `INSUFFICIENT_STOCK` | `UNAVAILABILITY_REASON_INSUFFICIENT_STOCK` |
| `PRODUCT_NOT_FOUND` | `UNAVAILABILITY_REASON_PRODUCT_NOT_FOUND` |

`AVAILABILITY_STATUS_UNSPECIFIED` is never returned for a successful application invocation.

---

## gRPC Error Mappings

| Exception type | gRPC status |
|---|---|
| `IllegalArgumentException` | `INVALID_ARGUMENT` |
| `NullPointerException` | `INVALID_ARGUMENT` |
| Any other `Exception` | `INTERNAL` |

Business unavailability (`INSUFFICIENT_STOCK`, `PRODUCT_NOT_FOUND`) maps to a successful RPC
response with `AVAILABILITY_STATUS_UNAVAILABLE` — never to a gRPC error status.

## Known Limitations

1. **No production `InventoryAvailabilityQueryPort` implementation.** The existing DB schema has
   only reservation tables. A stock-levels table and JPA adapter are required. This follows the
   same pattern as existing `WarehouseAllocationPort`/`StockAvailabilityPort` which also lack
   production implementations. Deferred to a future task.

2. **`net.devh:grpc-server-spring-boot-starter:3.1.0.RELEASE` targets Spring Boot 3.x.**
   It functions under Spring Boot 4.1.0 in testing, but a 4.x-aligned release should be
   preferred when available. Spring's own gRPC starter (`org.springframework.grpc`) is the
   long-term preferred option.

3. **`sun.misc.Unsafe` JDK warning.** Guava (transitive from gRPC) calls
   `Unsafe.objectFieldOffset` — logged at WARNING level, not a failure under Java 25. Will be
   addressed when Guava upgrades to non-Unsafe path (tracked in ADR-0007).

4. **gRPC reflection and health check.** `net.devh` auto-registers reflection and health services.
   These are visible in the Spring context log but not explicitly tested in this slice.
