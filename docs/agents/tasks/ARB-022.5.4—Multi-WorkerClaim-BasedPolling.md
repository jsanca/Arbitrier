Context

ARB-022.5.1 added CLAIMED status and lifecycle methods to OutboxEvent.
ARB-022.5.2 added claimEvent() and claimPending() to OutboxRepository and JpaOutboxRepositoryAdapter.
ARB-022.5.3 added SELECT ... FOR UPDATE SKIP LOCKED batch claiming to SpringDataOutboxRepository.

The polling runtime (OutboxPollingService, OutboxPollingScheduler, OutboxSchedulingAutoConfiguration)
still uses the non-claiming SequentialPendingDispatchService → findPending(limit) path.

Multiple workers polling concurrently would dispatch the same event multiple times.

This task wires the claim-based retrieval into the polling runtime so that concurrent
workers receive disjoint batches.

--------------------------------------------------

Scope

1. Create ClaimBasedBatchDispatchService
   - Calls claimPending(workerId, now, limit) instead of findPending(limit)
   - Dispatches claimed events sequentially via DispatchOutboxMessageService
   - Takes workerId as a constructor parameter

2. Update OutboxPollingService
   - Change delegate from SequentialPendingDispatchService to ClaimBasedBatchDispatchService
   - Call dispatchClaimed(batchSize) instead of dispatchPending(batchSize)

3. Update OutboxPollingProperties
   - Add worker-id property (default: blank = auto-generate from hostname + random suffix)

4. Restructure OutboxSchedulingAutoConfiguration
   - Condition on OutboxRepository + OutboundMessagePublisher (not SequentialPendingDispatchService)
   - Auto-configure DispatchOutboxMessageService, ClaimBasedBatchDispatchService,
     OutboxPollingService, OutboxPollingScheduler as a complete pipeline
   - Generate workerId from hostname + random suffix when property is blank

5. Tests
   - ClaimBasedBatchDispatchServiceTest (new) — unit tests for all dispatch behaviors
   - OutboxPollingServiceTest — update mock to ClaimBasedBatchDispatchService
   - OutboxSchedulingAutoConfigurationTest — update to test full pipeline wiring

--------------------------------------------------

Out of scope

- Stale claim recovery (CLAIMED rows that were never published) — ARB-022.6
- Retry / backoff — ARB-022.6
- SequentialPendingDispatchService remains (still tested, available for manual wiring if needed)
