## Summary

Kafka + Avro publishing foundation for `OrderCreatedDomainEvent`. Introduces topic naming constants
and Kafka header constants in `platform`, the `OrderCreated` Avro mapper, and
`KafkaOrderEventPublisher` in `order-service`. Domain and application layers remain Kafka/Avro-free.

## requestedTotal Decision (option a)

**Problem**: `OrderCreated` Avro schema has a required `requestedTotal: MoneyAmount` field.
`OrderCreatedDomainEvent` has no such field — source of truth for pricing is an open question
from ARB-006.

**Decision**: Emit `MoneyAmount("0", "USD")` as an explicit placeholder.

**Why option (a) over (b) and (c)**:
- Option (b) would require changing the `OrderEventPublisher` port interface and/or the command —
  premature without knowing where pricing will live.
- Option (c) deferring `KafkaOrderEventPublisher` entirely removes the primary deliverable.
- Option (a) is the smallest change. The placeholder is named and documented; the field is visible
  in tests, making it easy to track.

**Constraint**: Downstream consumers of `arbitrier.order.created.v1` **must not rely on
`requestedTotal`** until this open question is resolved.

OPEN QUESTION: Replace `MoneyAmount("0", "USD")` when a pricing source of truth is established
(catalog service, order domain model, or application-layer computation).

## correlationId in MessageMetadata

`OrderCreatedDomainEvent` does not yet carry a `correlationId`. `KafkaOrderEventPublisher` reads
`MDC.get("correlationId")` (populated by `CorrelationFilter` from `X-Correlation-Id` HTTP header)
and falls back to a fresh UUID if absent.

OPEN QUESTION: Thread `correlationId` through `SubmitCorporateBulkOrderCommand` → domain event →
Kafka message as a first-class value rather than an MDC side-channel. This is deferred to when
platform correlation is wired end-to-end.

## Kafka Header Strategy

| Header | Source | Status |
|--------|--------|--------|
| `messageId` | Generated by `OrderCreatedAvroMapper` (UUID per message) | Active |
| `correlationId` | Read from MDC; fallback: new UUID | Active |
| `schemaVersion` | Always `"v1"` | Active |
| `causationId` | Null for root messages — no upstream message | Not attached (null) |
| `traceparent` | W3C Trace Context — owned by OpenTelemetry SDK | Deferred |
| `tracestate` | W3C Trace Context — owned by OpenTelemetry SDK | Deferred |

W3C trace context injection into Kafka messages is deferred to the OpenTelemetry Kafka
instrumentation phase. B3 headers are not used (ADR-0008).

## Avro Serializer Decision (Deferred)

`KafkaPublisherConfiguration` currently configures `ByteArraySerializer` as the value serializer.
`KafkaOrderEventPublisher` passes the Avro `OrderCreated` object directly to `KafkaTemplate`.
**This will fail at runtime** unless the serializer is replaced before a real Kafka broker is used.

OPEN QUESTION: Choose the Avro serializer:
- **Confluent `KafkaAvroSerializer`** — requires Schema Registry; adds `io.confluent:kafka-avro-serializer` dependency.
- **Standalone `SpecificAvroSerializer`** — embeds schema in each message; no Schema Registry required at the cost of message overhead.

The production `KafkaPublisherConfiguration` must be updated with the chosen serializer and any
required properties (`schema.registry.url` etc.) before the first deployment.

## Activation Strategy (`@ConditionalOnProperty`)

`KafkaPublisherConfiguration` is annotated with `@ConditionalOnProperty("spring.kafka.bootstrap-servers")`.

- **Tests**: `spring.kafka.bootstrap-servers` is not set → `KafkaPublisherConfiguration` is skipped →
  `OrderServiceTestConfiguration` provides `RecordingOrderEventPublisher` (with `@Primary`) → no broker required.
- **Production**: `spring.kafka.bootstrap-servers=kafka:9092` → `KafkaPublisherConfiguration` creates
  `ProducerFactory`, `KafkaTemplate`, and `KafkaOrderEventPublisher`.

Note: Spring Boot 4.1.0 does not auto-configure `KafkaTemplate` (no `spring-boot-starter-kafka` auto-configuration).
All Kafka wiring is explicit in `KafkaPublisherConfiguration`.

## Native Image Considerations

OPEN QUESTION: The following items require `RuntimeHintsRegistrar` entries when building a native image:

| Area | Risk |
|------|------|
| Generated Avro classes (`OrderCreated`, `MessageMetadata`, etc.) | Avro uses reflection to write/read fields. Register all Avro-generated classes for reflection. |
| `SpecificDatumWriter` / `SpecificDatumReader` | Uses reflection over Avro schema field names. |
| `KafkaTemplate` | Spring's Kafka infrastructure is not currently registered for native-image. Validate with GraalVM `--verbose` trace. |
| `ByteArraySerializer` / value serializer | Instantiated by class name at runtime; must be registered for reflection. |

Do not add `RuntimeHints` until a native image build is attempted (future task per ADR-0007).

## Open Questions

1. **requestedTotal**: Replace `MoneyAmount("0", "USD")` placeholder when pricing source of truth is resolved.
2. **Avro serializer**: Choose between Confluent `KafkaAvroSerializer` and standalone Avro serializer; update `KafkaPublisherConfiguration` accordingly.
3. **correlationId propagation**: Thread `correlationId` through `SubmitCorporateBulkOrderCommand` → `OrderCreatedDomainEvent` → Avro metadata as a first-class value.
4. **W3C trace context in Kafka headers**: Wire OTel Kafka instrumentation to propagate `traceparent`/`tracestate` into producer records automatically.
5. **Native image**: Register Avro-generated classes and Kafka serializers when native image build is attempted.
